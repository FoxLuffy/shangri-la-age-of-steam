# Chat API & Engine Mapping Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Expose the Narrative Engine via a REST API endpoint for the frontend to consume.

**Architecture:** A POST `/chat` endpoint in FastAPI that accepts a `PlayerAction`, passes it to the `NarrativeEngine`, and returns a `NarrativeResult`.

**Tech Stack:** FastAPI, Pydantic, Python

---

### Task 1: Create Chat API Endpoint

**Objective:** Create a POST `/chat` endpoint that bridges the frontend and the engine.

**Files:**
- Modify: `shangri-la-age-of-steam/backend/main.py`

**Step 1: Implement Endpoint**
Update `main.py` to include the `/chat` route:
```python
from fastapi import FastAPI
from pydantic import BaseModel
from backend.models import PlayerAction, NarrativeResult
from backend.engine import NarrativeEngine
from backend.client import VLLMClient

app = FastAPI()

# This would typically be initialized via a dependency injection or global state
# For now, we'll instantiate them here or in a startup event
mock_client = VLLMClient(api_base="http://localhost:8000/v1")
# Note: In a production app, WorldState would be persisted in a DB
dummy_state = WorldState(
    current_location=Location(id="1", name="Forest", description="A dark forest"),
    active_npcs=[]
)
engine = NarrativeEngine(dummy_state, mock_client)

@app.get("/health")
async def health_check():
    return {"status": "ok"}

@app.post("/chat", response_model=NarrativeResult)
async def chat(action: PlayerAction):
    result = engine.process_action(action)
    return result
```

**Step 2: Verify with Test**
Create `backend/tests/test_api.py` to test the endpoint using `httpx.AsyncClient`.

**Step 3: Commit**
```bash
git add backend/
git commit -m "feat: add /chat endpoint"
```
